E.232. リリース7.4.18

<title>Release 7.4.18</title>

リリース日

<title>Release Date</title>

2007-09-17

This release contains fixes from 7.4.17. For information about new features in the 7.4 major release, see <xref linkend="release-7-4">. このリリースは7.4.17の不具合を修正したものです。 7.4メジャーリリースにおける新機能についてはE.250. リリース7.4を参照してください。

E.232.1. バージョン7.4.18への移行

<title>Migration to Version 7.4.18</title>

A dump/restore is not required for those running 7.4.X. However, if you are upgrading from a version earlier than 7.4.11, see <xref linkend="release-7-4-11">. 7.4.Xからの移行ではダンプ/リストアは不要です。 しかし、7.4.11より前のバージョンからアップグレードする場合は、E.239. リリース7.4.11を参照してください。

E.232.2. 変更点

<title>Changes</title>

  • Prevent index corruption when a transaction inserts rows and then aborts close to the end of a concurrent <command>VACUUM</> on the same table (Tom) トランザクションが、行を挿入した後に、同一テーブルに対する同時実行中のVACUUMの終了時刻とほぼ同じ時点でアボートした場合のインデックス破損を防止します。(Tom)

  • Make <command>CREATE DOMAIN ... DEFAULT NULL</> work properly (Tom) CREATE DOMAIN ... DEFAULT NULLが適切に動作するようにしました。(Tom)

  • Fix excessive logging of <acronym>SSL</> error messages (Tom) 過度なSSLエラーメッセージのログ処理を修正しました。(Tom)

  • Fix crash when <varname>log_min_error_statement</> logging runs out of memory (Tom) log_min_error_statementログ処理がメモリ不足になった時のクラッシュを修正しました。(Tom)

  • Prevent <command>CLUSTER</> from failing due to attempting to process temporary tables of other sessions (Alvaro) 他のセッションによる一時テーブル処理を防ぐために起こるCLUSTERの失敗を防ぎます。(Alvaro)

  • Require non-superusers who use <filename>/contrib/dblink</> to use only password authentication, as a security measure (Joe) セキュリティ強化のため、パスワード認証のみを使用する/contrib/dblinkを使用するユーザが非特権ユーザであることを要求します。(Joe)
