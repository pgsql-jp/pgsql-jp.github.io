E.1.1. 概要 #

《機械翻訳》PostgreSQL 17には、以下のような多くの新機能と拡張機能が含まれています。

  • TO BE COMPLETED LATER

《機械翻訳》上記の項目とPostgreSQL 17のその他の新機能については、以下の節で詳しく説明します。

E.1.2. バージョン17への移行 #

《機械翻訳》以前のリリースからデータを移行したい時は、どのリリースについても、pg_dumpallを利用したダンプとリストア、あるいはpg_upgradeやロジカルレプリケーションの使用が必要です。 新たなメジャーバージョンへの移行に関する一般的な情報については18.6を参照してください。

《機械翻訳》バージョン 17 には、以前のリリースとの互換性に影響する可能性のある多くの変更が含まれています。 次の互換性のない変更点に注意してください。

  • 《機械翻訳》メンテナンス操作中に安全なsearch_pathを使用するように関数を変更しました(Jeff Davis)。

    《機械翻訳》これにより、メンテナンス操作（ANALYZE、CLUSTER、REFRESH MATERIALIZED VIEW、REINDEX、またはVACUUM）が安全でないアクセスを実行しないようにします。 デフォルト以外のスキーマを参照する必要がある式索引およびマテリアライズド・ビューで使用されるファンクションは、ファンクション作成時に検索パスを指定する必要があります。

  • 《機械翻訳》間隔値の末尾にのみ「ago」を表示するように制限する(Joseph Koshakow)。

    《機械翻訳》また、空の間隔単位や間隔単位が複数回出現しないようにします。

  • 《機械翻訳》サーバ変数 old_snapshot_threshold を削除しました。 (Thomas Munro)

    《機械翻訳》この変数により、実行中のトランザクションにまだ表示されている可能性のある行を削除するために、vacuumが使用され、後でアクセスされた場合に「snapshot too old」エラーが発生する可能性がありました。 この機能は、改善された実装が見つかった場合、Postgresに後で再追加される可能性があります。

  • 《機械翻訳》サーバー変数 session_auth_is_superuser の名前を、より正確な現在のロール is_superuser に変更します (Nathan Bossart)。

  • 《機械翻訳》初期セッション・ユーザーのスーパーユーザー状態のSET SESSION AUTHORIZATION処理を変更する(Joseph Koshakow)。

    《機械翻訳》新しい動作は、接続時のユーザーのスーパーユーザー・ステータスではなく、SET SESSION AUTHORIZATIONコマンドが発行された時点でのセッション・ユーザーのスーパーユーザー・ステータスに基づいています。

  • 《機械翻訳》データベースごとのユーザをシミュレートする削除機能 (Nathan Bossart)

    《機械翻訳》この機能db_user_namespaceは、ほとんど使用されませんでした。

  • 《機械翻訳》wal_sync_method 値を削除する。 Windows の場合は fsync_writethrough (Thomas Munro)。

    《機械翻訳》この値はWindowsの"fsync"と同じである。

  • 《機械翻訳》2つのWALファイル名関数のファイル境界処理を変更しました。 (Kyotaro Horiguchi, Andres Freund, Bruce Momjian)

    《機械翻訳》以前は、LSNがファイルセグメント境界上にある場合、pg_walfile_name()とpg_walfile_name_offset()は前のLSNセグメント番号を報告していました。 現在はLSNセグメントを返します。

  • 《機械翻訳》サーバー変数の trace_recovery_messages は、もう必要ないため削除します (Bharath Rupireddy)。

  • 《機械翻訳》informationスキーマ列のelement_types.domain_defaultを削除しました。 (Peter Eisentraut)

  • 《機械翻訳》pgrowlocks ロックモードの出力ラベルを変更しました。 (Bruce Momjian)

  • 《機械翻訳》名前変更 pg_stat_statementのI/Oブロック読み取り/書き込みタイミング統計カラム(Nazir Bilal Yavuz)

    《機械翻訳》これにより、"blk_read_time" は "shared_blk_read_time" に、"blk_write_time" は "shared_blk_write_time" に名前が変更されます。

  • 《機械翻訳》pg_stat_checkpointerからbuffers_backendとbuffers_backend_fsyncを削除しました。 (Bharath Rupireddy)

    《機械翻訳》これらのフィールドはpg_stat_ioの同様の列と重複していると見なされます。

  • 《機械翻訳》pg_attribute.attstattargetとpg_attribute.stxstattargetを変更して、デフォルトの統計ターゲットをNULLとするようにしました。 (Peter Eisentraut)

  • 《機械翻訳》pg_stat_progress_vacuumの列max_dead_tuplesをmax_dead_tuple_bytesに、num_dead_tuplesをdead_tuple_byteに変更しました（佐和田雅彦）。

    《機械翻訳》これらの列は、タプルではなくバイトを報告するようになりました。

  • 《機械翻訳》システムビューpg_stat_slru内のSLRU列の名前を変更しました。 (Alvaro Herrera)

    《機械翻訳》pg_stat_slru_rest() が受け付けるカラム名も変更されました。

E.1.3.1.1. オプティマイザ #

  • 《機械翻訳》以前の行出力節で参照された列の統計を考慮することによって、CTEプランを改善できるようにオプティマイザを許可する(Jian Guo, Tom Lane)。

  • 《機械翻訳》以前の行出力句で参照された列のソート順を考慮することで、CTE計画を改善できるようにオプティマイザを許可します(Richard Guo)。

  • 《機械翻訳》IS NOT NULLとIS NULL問い合わせ制限の最適化を改善しました。 (David Rowley, Richard Guo, Andy Fan)

    《機械翻訳》IS NOT NULL クエリの制限を NOT NULL 列から削除し、IS NULL が指定された場合に NOT NULL 列のスキャンを排除します。

  • 《機械翻訳》IS [NOT] UNKNOWN条件のブール列に対するパーティション・プルーニングを許可(David Rowley)

  • 《機械翻訳》包含演算子<@および@>を使用する場合の範囲値の最適化を改善しました。 (Kim Johan Andersson, Jian He)

  • 《機械翻訳》クエリノードの並列実行を可能にする (Tom Lane)

  • 《機械翻訳》GROUP BY 列を ORDER BY に合わせて内部的にソートできるようにしました。 (Andrei Lepikhov, Teodor Sigaev)

    《機械翻訳》これは、サーバ変数 enable_group_by_reordering を使用して無効にできます。

  • 《機械翻訳》UNION （ALL を使用しない） で MergeAppend を使用できるようにしました。 (David Rowley)

  • 《機械翻訳》マージアペンローズ氏は、ソートする必要のある行数をより正確に計算するFix MergeAppendを計画している（Alexander Kuzmenkov氏）。

  • 《機械翻訳》GiSTインデックスとSP-GiSTインデックスを増分ソートの一部にできるようにしました。 (Miroslav Bendik)

    《機械翻訳》これは、最初のカラムが GiST インデックスと SP-GiST インデックスを持ち、他のカラムが持たない ORDER BY 句に特に有効です。

  • 《機械翻訳》pg_statsに列を追加して、範囲型ヒストグラム情報を報告する(Egor Rogov, Soumyadeep Chakraborty)。

E.1.3.1.4. 監視 #

  • 《機械翻訳》システムビューpg_stat_checkpointerの作成（Bharath Rupireddy、Anton A. Melnikov、Alexander Korotkov）

    《機械翻訳》pg_stat_bgwriterから関連する列が削除され、この新しいシステムビューに追加されました。

  • 《機械翻訳》pg_stat_reset_shared() で全ての共有統計情報をリセットできるようにしました。 （Torikoshi 敦）

    《機械翻訳》これは NULL を渡すことで行われます。

  • 《機械翻訳》pg_stat_reset_shared("slru")でSLRUの統計をクリアできるようにしました。 (Torikoshi Atsushi)

    《機械翻訳》また、pg_stat_reset_shared(NULL)はSLRUの統計情報もリセットします。

  • 《機械翻訳》pg_stat_reset_slru() がすべてのSLRU統計をリセットできるようにしました。 (Bharath Rupireddy)

    《機械翻訳》コマンドpg_stat_reset_slru(NULL)はすでにこれを行いました。

  • 《機械翻訳》バックアップからのWAL回復に関連するログメッセージを追加しました。 (Andres Freund)

  • 《機械翻訳》"trust" 接続の log_connections ログ行を追加しました。 (Jacob Champion)

  • 《機械翻訳》レポートにログメッセージを追加する (Bharath Rupireddy)

    《機械翻訳》これは、サーバ変数log_replication_commandsによって有効になります。

  • 《機械翻訳》同じクエリに対して他のログレコードと同じ時間値を使用するように、emit_log_hookを修正しました。 (Kambam Vinay, Michael Paquier)

  • 《機械翻訳》待機イベントの型を報告するシステムビューpg_wait_eventsを追加しました。

    《機械翻訳》これは、pg_stat_activityで報告される待機イベントに説明を追加する場合に便利です。

  • 《機械翻訳》チェックポイント遅延の待機イベントの追加(Thomas Munro)

  • 《機械翻訳》真空を許す インデックス処理の進行状況を報告する （サミ・イムシェル）。

    《機械翻訳》これは、システムビューの列pg_stat_progress_vacuum.indexes_totalとpg_stat_progress_vacuum.indexes_processedに表示されます。

  • 《機械翻訳》接続の暗号化をデバッグできるように、サーバ変数 trace_connection_negotiation を追加しました。 (Heikki Linnakangas, Kyotaro Horiguchi)

E.1.3.1.5. 権限 #

  • 《機械翻訳》保守操作を制御するために、テーブルごとにGRANT権限MAINTAINを追加しました(Nathan Bossart)。

    《機械翻訳》操作は、VACUUM、ANALYZE、REINDEX、REFRESH MATERIALIZED VIEW、CLUSTER、LOCK TABLEです。

  • 《機械翻訳》ユーザが付与できるロールを追加しました。 メンテナンス操作を制御します。 (Nathan Bossart)

    《機械翻訳》操作は、VACUUM、ANALYZE、REINDEX、REFRESH MATERIALIZED VIEW、CLUSTER、LOCK TABLEです。

  • 《機械翻訳》pg_monitorメンバーシップを持つロールがpg_current_logfile()を実行できるようにしました。 (Pavlo Golub, Nathan Bossart)

E.1.3.1.6. サーバ設定 #

  • 《機械翻訳》システム変数 allow_alter_system を追加して、ALTER SYSTEM を禁止する （Jelte Fennema-Nio、Gabriele Bartolini）

  • 《機械翻訳》ALTER SYSTEMで認識されないカスタム・サーバ変数を設定できるようにしました。 (Tom Lane)

    《機械翻訳》これは、GRANT ON PARAMETERでも可能です。

  • 《機械翻訳》トランザクションの持続時間を制限するために、サーバ変数 transaction_timeout を追加します （Andrey Borodin、Japin Li、Junwang Zhao、Alexander Korotkov）。

  • 《機械翻訳》libc の C ロケールと同様の「組み込み」照合順序プロバイダを作成する (Jeff Davis)。

    《機械翻訳》これはlibcと同じですが、独立した"C"ロケールを使用します。 しかし、"C"ロケールでは"C"以外の照合順序（例えば"en_US"や"C.UTF-8"）を使用することができます。 詳細

  • 《機械翻訳》サーバ変数 huge_pages_status を追加して、Postgres による巨大ページの使用状況を報告する (Justin Pryzby)

    《機械翻訳》これは、 huge_pages が "try" に設定されている場合に便利です。

  • 《機械翻訳》イベント・トリガーを無効にするためのサーバ変数を追加しました。 (Daniel Gustafsson)

    《機械翻訳》この設定 event_triggers は、デバッグのためにイベント・トリガーを一時的に無効にすることができます。

  • 《機械翻訳》SLRUキャッシュサイズを設定できるようにする(Andrey Borodin, Dilip Kumar)。

    《機械翻訳》新しいサーバ変数は、commit_timestamp_buffers、multixact_member_buffers、multixact_offset_buffers、notify_buffers、serializable_buffers、subtransaction_buffers、およびtransaction_buffersです。

E.1.3.1.7. Streaming Replication and Recovery #

  • 《機械翻訳》ファイルシステムの増分バックアップのサポートを追加しました。

    《機械翻訳》増分バックアップは、pg_basebackupの新しい--incrementalオプションを使用して作成できます。 新しいアプリケーションpg_combinebackupは、ファイルシステムの基本バックアップと増分バックアップの操作を可能にします。

  • 《機械翻訳》WAL要約ファイルの作成を許可する(Robert Haas, Nathan Bossart, Hubert Depesz Lubaczewski)。

    《機械翻訳》これらのファイルは、LSN範囲内で変更されたブロック番号を記録し、増分ファイルシステムバックアップに有用です。 これは、サーバ変数 summarize_wal と wal_summarize_keep_time によって制御され、pg_available_wal_summaries()、pg_wal_summary_contents()、および pg_get_wal_summarizer_state() でイントロスペクトされます。

  • 《機械翻訳》システム識別子をファイルシステムバックアップマニフェストファイルに追加する (Amul Sul)。

    《機械翻訳》これは、無効なWAL使用を検出するのに役立ちます。

  • 《機械翻訳》pg_basebackupがpostgresql.auto.confに接続情報を書き込む際に、接続文字列値dbnameを書き込めるようにしました。 (Vignesh C, Hayato Kuroda)

  • 《機械翻訳》スロットが無効な理由を報告するための列pg_replication_slots.invalidation_reasonの追加 (Shveta Malik, Bharath Rupireddy)

  • 《機械翻訳》スロットの非アクティブ期間を報告するためにpg_replication_slots.inactive_sinceを追加しました。 (Bharath Rupireddy)

  • 《機械翻訳》論理レプリケーションスロットを同期するための関数pg_sync_replication_slots()を追加しました。 (Hou Zhijie, Shveta Malik, Ajin Cherian, Peter Eisentraut)

  • 《機械翻訳》フェイルオーバー・プロパティをレプリケーション・プロトコルに追加する（Hou Zhijie、Shveta Malik）。

E.1.3.1.8. Logical Replication #

  • 《機械翻訳》物理スタンバイサーバから論理レプリカを作成するためのアプリケーションpg_createsubscriberを追加しました。 (Euler Taveira)

  • 《機械翻訳》pg_upgradeが有効な論理スロットとサブスクリプションを移行するようにしました。 (Hayato Kuroda, Hou Zhijie, Vignesh C, Julien Rouhaud, Shlok Kyal)

    《機械翻訳》これにより、アップグレード後も論理レプリケーションを迅速に続行できます。 これは、PostgreSQLバージョン17.0以降の古いクラスタでのみ動作します。

  • 《機械翻訳》論理スロットのフェイルオーバーを有効にする（Hou Zhijie、Shveta Malik、Ajin Cherian）。

    《機械翻訳》これはpg_create_logical_replication_slot()のオプションの5番目の引数で制御します。

  • 《機械翻訳》フェイルオーバー論理スロット同期を有効にするために、サーバ変数sync_replication_slotsを追加します（Shveta Malik、Hou Zhijie、Peter Smith）。

  • 《機械翻訳》CREATE/ALTER SUBSCRIPTIONに論理レプリケーション・フェイルオーバー制御を追加（Shveta Malik、Hou Zhijie、Ajin Cherian）

  • 《機械翻訳》サブスクライバでハッシュ索引を使用するために、論理レプリケーションの変更を適用できるようにします(Kuroda)。

    《機械翻訳》以前は、この目的で使用できるのはbtreeインデックスだけでした。

  • 《機械翻訳》サブスクリプション所有者のスーパーユーザ権限が取り消された場合、適用ワーカーを再起動する (Vignesh C)。

    《機械翻訳》これにより、再認証が強制されます。

  • 《機械翻訳》pg_logical_emit_message()に"flush"オプションを追加しました。 (Michael Paquier)

    《機械翻訳》これにより、メッセージが永続化されます。

  • 《機械翻訳》サブスクライバに表示される前に同期化する必要がある物理スタンバイを指定できるようにします（Hou Zhijie、Shveta Malik）。

    《機械翻訳》サーバ変数は standby_slot_names です。

  • 《機械翻訳》pg_stat_subscriptionにworker type列を追加しました。 (Peter Smith)

E.1.3.2. ユーティリティコマンド #

  • 《機械翻訳》新しいCOPYオプション"ON_ERROR ignore"を追加してエラー行を破棄する(Damir Belyalov, Atsushi Torikoshi, Alex Shulgin, Jian He, Yugo Nagata)。

  • 《機械翻訳》EXPLAIN でオプティマイザのメモリ使用状況を報告できるようにしました。 (Ashutosh Bapat)

    《機械翻訳》このオプションは「MEMORY」と呼ばれます。

  • 《機械翻訳》ネットワーク転送のためのデータ変換コストを報告するために、EXPLAINオプションSERIALIZEを追加しました。 (Stepan Rutz, Matthias van de Meent)

  • 《機械翻訳》pg_stat_statementsに格納された2フェーズコミットのGIDをプレースホルダで置き換えました。 (Michael Paquier)

    《機械翻訳》これにより、PREPARE TRANSACTION、COMMIT PREPARED、ROLLBACK PREPARED を記録するために必要なエントリの数が大幅に減少します。

  • 《機械翻訳》pg_stat_statements内のDEALLOCATEを追跡（Dagfinn Ilmari Mannsåker, Michael Paquier）。

    《機械翻訳》DEALLOCATE名はpg_stat_statements内でプレースホルダとして保存されます。

  • 《機械翻訳》ローカルI/Oブロックの読み取り/書き込みタイミング統計を追加 pg_stat_statementの列(Nazir Bilal Yavuz)

    《機械翻訳》新しい列は、「local_blk_read_time」と「local_blk_write_time」です。

  • 《機械翻訳》JIT deform_counterの詳細をpg_stat_statementsに追加しました。 (Dmitry Dolgov)

  • 《機械翻訳》pg_stat_statements_reset()にオプションの4番目のパラメータを追加して、最小/最大統計のみをリセットできるようにしました。 (Andrei Zubkov)

    《機械翻訳》このパラメータのデフォルト値は「false」です。

  • 《機械翻訳》pg_stat_statements列に"stats_since"と"minmax_stats_since"を追加して、エントリの作成時間と最後の最小/最大リセット時間を追跡するようにしました。 (Andrei Zubkov)
