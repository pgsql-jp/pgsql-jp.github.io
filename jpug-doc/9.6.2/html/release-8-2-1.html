E.179. リリース8.2.1

<title>Release 8.2.1</title>

リリース日

<title>Release Date</title>

2007-01-08

This release contains a variety of fixes from 8.2. For information about new features in the 8.2 major release, see <xref linkend="release-8-2">. このリリースは8.2の各種不具合を修正したものです。 8.2メジャーリリースにおける新機能についてはE.180. リリース8.2を参照してください。

E.179.1. バージョン8.2.1への移行

<title>Migration to Version 8.2.1</title>

A dump/restore is not required for those running 8.2. 8.2からの移行ではダンプ/リストアは不要です。

E.179.2. 変更点

<title>Changes</title>

  • Fix crash with <literal>SELECT</> ... <literal>LIMIT ALL</> (also <literal>LIMIT NULL</>) (Tom) SELECT ... LIMIT ALL（またはLIMIT NULL）時のクラッシュを修正しました。(Tom)

  • <filename>Several /contrib/tsearch2</> fixes (Teodor) /contrib/tsearch2に対する複数の修正(Teodor)

  • On Windows, make log messages coming from the operating system use <acronym>ASCII</> encoding (Hiroshi Saito) Windowsにおいて、オペレーティングシステムからのログメッセージをASCII符号化方式に変更しました。(Hiroshi Saito)

    This fixes a conversion problem when there is a mismatch between the encoding of the operating system and database server. この修正は、オペレーティングシステムとデータベースサーバの間で符号化方式に違いがある場合の変換問題を解消するものです。

  • Fix Windows linking of <application>pg_dump</> using <filename>win32.mak</> Windowsにおけるwin32.makを使用したpg_dumpのリンクを修正しました。 (Hiroshi Saito)

  • Fix planner mistakes for outer join queries (Tom) 外部結合問い合わせに対するプランナの間違いを修正しました。(Tom)

  • Fix several problems in queries involving sub-SELECTs (Tom) 副問い合わせを含む問い合わせにおける複数の問題を修正しました。(Tom)

  • Fix potential crash in SPI during subtransaction abort (Tom) SPIにおける副トランザクションのアボート時のクラッシュの可能性を修正しました。(Tom)

    This affects all PL functions since they all use SPI. PLはすべてSPIを使用しますので、これはすべてのPL関数に影響します。

  • Improve build speed of <acronym>PDF</> documentation (Peter) PDF文書作成を高速化しました。(Peter)

  • Re-add <acronym>JST</> (Japan) timezone abbreviation (Tom) JST (日本)時間帯省略形を再度追加しました。(Tom)

  • Improve optimization decisions related to index scans (Tom) インデックススキャンに関連した最適化方針の決定を改良しました。(Tom)

  • Have <application>psql</> print multi-byte combining characters as before, rather than output as <literal>\u</> (Tom) psqlにおいて、複数バイトを組み合わせた文字を\uとして出力せずに、以前のように出力するようにしました。(Tom)

  • Improve index usage of regular expressions that use parentheses (Tom) 括弧を使用した正規表現を使用するインデックスを改良しました。(Tom)

    This improves <application>psql</> <literal>\d</> performance also. この改良により、psql \dの性能も向上しました。

  • Make <application>pg_dumpall</> assume that databases have public <literal>CONNECT</> privilege, when dumping from a pre-8.2 server (Tom) 8.2以前のサーバからダンプする際に、pg_dumpallは、そのデータベースがPUBLICにCONNECT権限を持たせていることを前提とするようにしました。(Tom)

    This preserves the previous behavior that anyone can connect to a database if allowed by <filename>pg_hba.conf</>. これは、pg_hba.confで許可されていれば、誰でもデータベースに接続できるという以前の振る舞いを維持します。
