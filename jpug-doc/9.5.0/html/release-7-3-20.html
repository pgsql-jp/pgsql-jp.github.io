E.232. リリース7.3.20

<title>Release 7.3.20</title>

リリース日

<title>Release Date</title>

2007-09-17

This release contains fixes from 7.3.19.
このリリースは7.3.19の不具合を修正したものです。

E.232.1. バージョン7.3.20への移行

<title>Migration to Version 7.3.20</title>

A dump/restore is not required for those running 7.3.X. However, if you are upgrading from a version earlier than 7.3.13, see <xref linkend="release-7-3-13">.
7.3.Xからの移行ではダンプ/リストアは不要です。 しかし、7.3.13より前のバージョンからアップグレードする場合は、「リリース7.3.13」を参照してください。

E.232.2. 変更点

<title>Changes</title>

  • Prevent index corruption when a transaction inserts rows and then aborts close to the end of a concurrent <command>VACUUM</> on the same table (Tom)
    トランザクションが、行を挿入した後に、同一テーブルに対する同時実行中のVACUUMの終了時刻とほぼ同じ時点でアボートした場合のインデックス破損を防止します。(Tom)

  • Make <command>CREATE DOMAIN ... DEFAULT NULL</> work properly (Tom)
    CREATE DOMAIN ... DEFAULT NULLが適切に動作するようにしました。(Tom)

  • Fix crash when <varname>log_min_error_statement</> logging runs out of memory (Tom)
    log_min_error_statementログ処理がメモリ不足になった時のクラッシュを修正しました。(Tom)

  • Require non-superusers who use <filename>/contrib/dblink</> to use only password authentication, as a security measure (Joe)
    セキュリティ強化のため、パスワード認証のみを使用する/contrib/dblinkを使用するユーザが非特権ユーザであることを要求します。(Joe)
