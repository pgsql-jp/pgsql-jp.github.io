E.163. リリース8.1.9

<title>Release 8.1.9</title>

リリース日

<title>Release Date</title>

2007-04-23

This release contains a variety of fixes from 8.1.8, including a security fix. For information about new features in the 8.1 major release, see <xref linkend="release-8-1">.
このリリースは8.1.8の各種不具合を修正したもので、セキュリティ問題の修正を1つ含みます。 8.1メジャーリリースにおける新機能については「リリース8.1」を参照してください。

E.163.1. バージョン8.1.9への移行

<title>Migration to Version 8.1.9</title>

A dump/restore is not required for those running 8.1.X. However, if you are upgrading from a version earlier than 8.1.2, see <xref linkend="release-8-1-2">.
8.1.Xからの移行ではダンプ/リストアは不要です。 しかし、8.1.2より前のバージョンからアップグレードする場合は、「リリース8.1.2」を参照してください。

E.163.2. 変更点

<title>Changes</title>

  • Support explicit placement of the temporary-table schema within <varname>search_path</>, and disable searching it for functions and operators (Tom)
    search_pathにおける一時テーブルスキーマの明示的な指定をサポートしました。 また、関数や演算子ではその検索を無効にしました(Tom)。

    This is needed to allow a security-definer function to set a truly secure value of <varname>search_path</>. Without it, an unprivileged SQL user can use temporary objects to execute code with the privileges of the security-definer function (CVE-2007-2138). See <command>CREATE FUNCTION</> for more information.
    これは、SECURITY DEFINER関数がsearch_pathの値を本当に安全に設定するために必要です。 こうしないと、一時オブジェクトを使用して権限のないSQLユーザがSECURITY DEFINER関数の権限でコードを実行することができてしまうためです（CVE-2007-2138）。 詳細はCREATE FUNCTIONを参照してください。

  • <filename>/contrib/tsearch2</> crash fixes (Teodor)
    /contrib/tsearch2のクラッシュを修正しました(Teodor)。

  • Require <command>COMMIT PREPARED</> to be executed in the same database as the transaction was prepared in (Heikki)
    COMMIT PREPAREDは、そのトランザクションを準備したデータベースと同じデータベースで実行する必要があります。(Heikki)

  • Fix potential-data-corruption bug in how <command>VACUUM FULL</> handles <command>UPDATE</> chains (Tom, Pavan Deolasee)
    VACUUM FULLのUPDATE連鎖方法において潜在したデータ破損不具合を修正しました(Tom, Pavan Deolasee)。

  • Planner fixes, including improving outer join and bitmap scan selection logic (Tom)
    外部結合の向上やビットマップスキャンの選択ロジックなど、プランナを修正しました(Tom)。

  • Fix PANIC during enlargement of a hash index (bug introduced in 8.1.6) (Tom)
    ハッシュインデックスの拡張時のパニックを修正しました(8.1.6で入ってしまった不具合です)(Tom)。

  • Fix POSIX-style timezone specs to follow new USA DST rules (Tom)
    POSIX書式の時間帯指定が新しいUSA DST規則に従うよう修正しました(Tom)。
