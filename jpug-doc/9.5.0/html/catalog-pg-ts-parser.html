49.54. pg_ts_parser

<para> The <structname>pg_ts_parser</structname> catalog contains entries defining text search parsers. A parser is responsible for splitting input text into lexemes and assigning a token type to each lexeme. Since a parser must be implemented by C-language-level functions, creation of new parsers is restricted to database superusers. </para>

pg_ts_parserカタログはテキスト検索パーサを定義する項目を含みます。 パーサは、入力テキストを字句に分割することとトークン型を字句に割り当てることに責任を持ちます。 パーサはC言語レベルの関数で実装されていなくてはいけないため、新規のパーサの作成はデータベースのスーパーユーザに制限されています。

<para> <productname>PostgreSQL</productname>'s text search features are described at length in <xref linkend="textsearch">. </para>

PostgreSQLのテキスト検索機能については12章全文検索で詳しく説明します。
